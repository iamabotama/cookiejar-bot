How It Works
Understand the multisig flow
1. Create a multisig with members, permissions, threshold, and optional authorities.
2. Fund a vault PDA and create proposals for transfers or configuration changes.
3. Members approve or reject proposals based on their voting permission.
4. Once a proposal is approved and any time lock has passed, an executor can run it.
What You Can Do
Core capabilities included
Create autonomous or controlled multisigs.
View members, permissions, vault balances, spending limits, and proposals.
Create vault transfer proposals and config proposals.
Approve, reject, cancel, activate, and execute proposals.
Use existing spending limits directly from the UI.
